## ✨ Features

- 🔐 **JWT Authentication** — Secure register / login with bcrypt password hashing
- 💬 **Real-time AI Streaming** — WebSocket-based token-by-token AI response via OpenAI API
- 📁 **Conversation Management** — Full CRUD for chat sessions with history persistence
- 🗄️ **Optimized MySQL Schema** — Indexed queries for fast message retrieval
- 📄 **Auto API Docs** — Swagger UI & ReDoc built-in via FastAPI
- 🐳 **Dockerized** — One-command local setup with Docker Compose
- ⚙️ **CI/CD Pipeline** — Automated testing and deployment via GitHub Actions → AWS EC2

## ⚡ Quick Start

### Prerequisites
- Docker & Docker Compose installed
- OpenAI API Key

### Run Locally

```bash
# 1. Clone the repo
git clone https://github.com/yourname/ai-chat-platform.git
cd ai-chat-platform

# 2. Set up environment variables
cp .env.example .env
# Fill in your OPENAI_API_KEY and DB credentials

# 3. Start with Docker Compose

# 4. API running at
# http://localhost:8000
# http://localhost:8000/docs  ← Swagger UI
```

## 📡 API Reference

> Full interactive docs available at `/docs` after running the server.

### Auth
```
POST /api/auth/register   # Create account
POST /api/auth/login      # Login, returns JWT
```

### Chat  *(Authorization: Bearer <token>)*
```
GET    /api/chat          # List all conversations
POST   /api/chat          # Create new conversation
DELETE /api/chat/{id}     # Delete conversation
```

### Messages  *(Authorization: Bearer <token>)*
```
GET  /api/messages/{conversation_id}   # Get chat history
POST /api/messages/{conversation_id}   # Send message → triggers AI stream
```

### WebSocket
```
WS /ws/{conversation_id}?token=<JWT>

# Server sends:
{ "type": "stream_chunk", "content": "Hello" }
{ "type": "stream_end",   "message_id": 42 }
{ "type": "error",        "detail": "..." }
```

## 🗄️ Database Design

```sql
users           → id, username, email, hashed_password, created_at
conversations   → id, user_id (FK), title, created_at
messages        → id, conversation_id (FK), role ENUM('user','assistant'), content, created_at
```

**Optimization：**
- Index on `messages.conversation_id` for fast history queries
- Index on `conversations.user_id` for fast user session lookup
